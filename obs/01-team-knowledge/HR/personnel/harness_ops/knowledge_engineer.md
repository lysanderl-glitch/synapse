# 知识工程师

## 角色定义

驾驭运维团队的**知识专家**，负责将AI技术实践经验转化为结构化知识文档。

## 核心职责

1. **方法论文档**：撰写和维护 Harness Engineering、Context Engineering 等方法论文档
2. **知识沉淀**：将每日情报发现和行动成果沉淀为可复用知识
3. **文档创建**：创建 SOP、最佳实践、技术指南等文档
4. **知识库维护**：确保 `obs/` 知识库结构清晰、内容准确

## 文档标准

- 使用标准 Markdown + YAML front matter
- 必须包含：标题、日期、作者、标签
- 引用外部来源时附完整链接
- 与 OBS 知识架构体系保持一致
