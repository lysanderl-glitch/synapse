# 决策体系说明

## 核心理念

原则是"规则"而非"建议"——通过代码级强制执行，确保决策原则被遵守。

## 决策层级

```
┌────────────────────────────────────────┐
│          总裁（用户）                    │
│     战略/组织/重大投资 最终决策         │
└────────────────────────────────────────┘
                  ↑
                  │ 上报
                  │
┌────────────────────────────────────────┐
│         Lysander CEO (AI)              │
│     日常管理/团队协调/方案审批         │
└────────────────────────────────────────┘
                  ↑
                  │ 召集
                  │
┌────────────────────────────────────────┐
│            智囊团 Graphify               │
│      分析/洞察/趋势/决策支持           │
└────────────────────────────────────────┘
```

## 决策类型

### 1. small_problem（小问题）

**判断条件**：
- 风险可控
- 有明确执行路径
- 不影响核心架构

**关键词**：
```python
["同步", "sync", "生成yaml", "加载", "查询",
 "显示", "列出", "查看", "获取状态",
 "组装团队", "召唤", "路由"]
```

**处理**：直接执行，无需询问

### 2. require_code_review（代码审计）

**判断条件**：
- 涉及代码/脚本编写
- 需要QA检查

**关键词**：
```python
["脚本", "代码", "实现", "修改代码", "编写",
 "harness", "workflow", "自动化脚本"]
```

**处理**：触发代码审计流程后方可执行

### 3. think_tank（智囊团决策）

**判断条件**：
- 涉及策略/方案/流程变更
- 需要深度分析

**关键词**：
```python
["新架构", "新方案", "策略调整", "原则修改",
 "自动化方案", "决策体系", "流程变更",
 "分析", "战略", "洞察"]
```

**处理**：召集智囊团分析，决策后执行

### 4. escalate（上报总裁）

**判断条件**：
- 超出AI决策授权
- 涉及战略/组织/重大投资

**关键词**：
```python
["战略", "裁员", "重大投资", "组织调整"]
```

**处理**：上报总裁决策

## 决策流程

```
用户输入任务
    ↓
decision_check(task)
    ↓
┌────────────────────────────────────────┐
│  [1] 小问题？                          │
│      是 → 返回 {decision: "small_problem"} │
│      否 → 继续                         │
├────────────────────────────────────────┤
│  [2] 代码审计？                         │
│      是 → 返回 {decision: "require_code_review"} │
│      否 → 继续                         │
├────────────────────────────────────────┤
│  [3] 智囊团决策？                       │
│      是 → 返回 {decision: "think_tank"} │
│      否 → 继续                         │
├────────────────────────────────────────┤
│  [4] 超出授权？                         │
│      是 → 返回 {decision: "escalate"}  │
│      否 → 返回 {decision: "think_tank"} │
└────────────────────────────────────────┘
```

## 执行链机制

任务完成后自动执行后续步骤，消除条件反射式询问：

```python
TASK_EXECUTION_CHAIN = {
    "同步OBS人员卡片到YAML": ["构建网站"],
    "构建网站": ["发布博客文章"],
    "发布博客文章": [],
}
```

## 最佳实践

1. **明确任务描述**：包含关键词便于系统判断
2. **避免模糊请求**：如"帮我看看"应改为"查看同步状态"
3. **遵循角色定位**：总裁指令 vs 日常任务
