产品定位

Synapse Forge 是从 Janus Digital 内部 AI 运营体系中提炼而来的开源产品，将经过实战验证的 Multi-Agent 管理方法论，锻造成任何团队都能使用的标准化工具。

命名与定位

Synapse（突触）— 神经元之间传递信号的关键节点。知识 ↔ 决策 ↔ 执行，信息流转的核心枢纽。

Forge（锻造）— 将原始金属锻造成精密工具的过程。Synapse Forge 将 AI Agent 的原始能力，锻造成结构化、可管理、可扩展的 AI 团队。

一句话定位：从内部工具到开源产品，让每个团队都能"锻造"自己的 AI 团队。

"Synapse 不是又一个 Agent 框架 — 它是第一个 AI 团队管理操作系统"

USP 三支柱

⚙

Harness Engineering

Agent = Model + Harness。通过 Guides（前馈控制）、Workflow（结构化流程）、Constraints（约束系统）三层治理，让 AI Agent 在框架内高效运转。

区别于传统 prompt engineering 的碎片化方式。

★

管理层，非技术层

CEO → 智囊团 → 执行团队的三层管理体系。四级决策制度（L1自动 → L4总裁）。让 AI 像真正的团队一样运作。

面向业务管理者，而非开发者。

⚡

零代码到可运行

synapse init 一键生成完整配置。预设模板覆盖常见团队场景。无需理解底层原理，选择模板即可启动。

5分钟内从零到拥有 AI 团队。

体系架构

四层模块化架构，从核心引擎到扩展生态，每一层独立可用、逐层增值。

Core Layer — 核心层（必装）
Harness Engine + Agent HR + 决策体系 + 执行链
Business Layer — 业务层（按需选装）
情报管线 + 内容运营 + 研发工程 + 增长引擎
Support Layer — 支撑层（推荐）
知识管理 OBS + Butler 管家 + 自动化编排
Extension Layer — 扩展层（可选）
SPE 个人引擎 + 自定义模块 + 社区插件

并行运行策略

ai-team-system（内部实例）与 synapse-core（产品）并行运行，逐步迁移。

维度 ai-team-system（现状） synapse-core（目标）
定位 Janus Digital 内部运营实例 通用产品，任何团队可用
CLAUDE.md ~25,000 tokens（单体配置） ~9,400 tokens（模块化组装）
Agent 管理 44 人固定编制 模块化按需选配
配置方式 手动编辑 YAML/MD synapse init 交互式生成
扩展性 Janus Digital 专用 模块市场 + 社区贡献
仓库 Private（ai-team-system） Public（synapse-core）

切换决策点

继续并行的条件

  • ● synapse-core 功能未完全覆盖 ai-team-system
  • ● 迁移风险未充分评估
  • ● 内部实例有特殊定制需求

完全切换的条件

  • ✓ synapse-core 功能 ≥ ai-team-system
  • ✓ 迁移脚本经过充分测试
  • ✓ 种子用户反馈正面（满意度 ≥ 4.0/5）
  • ✓ 所有自动化管线已适配新架构
